🔒 Private S3 + Temporary Links

Corporate event photos, securely delivered.

Photos are stored in a private Amazon S3 bucket. When an employee clicks View Image, the app asks a serverless backend (API Gateway + Lambda) to create a temporary 5-minute pre-signed URL.

This means you can’t open S3 objects directly, and URLs automatically expire.

Quick How It Works

  1. Pick an image in the Gallery.
  2. Frontend calls API Gateway.
  3. Lambda checks the file exists in S3.
  4. Lambda returns a pre-signed URL valid for 300 seconds.
  5. Browser displays the image using that temporary URL.

✨ About

PixelWave Marketing Image Gallery

This is a beginner-friendly serverless demo that connects a simple multi-page HTML UI with a secure backend. Images remain private in S3. The only access path is a Lambda generated pre-signed URL that expires after 5 minutes.

Stack: Amplify (hosting) + API Gateway (REST) + Lambda (Python 3.9) + S3 (private).

📩 Contact

Need access help?

For this challenge, the gallery is designed to work for employees. If you’re setting this up, verify:

  • Lambda has permission to read from your S3 bucket.
  • API Gateway has CORS enabled.
  • You updated API_URL in index.html with your endpoint.
  • Your S3 bucket blocks public access.
